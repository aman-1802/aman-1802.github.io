SaudaaAI — Voice to Quotation, Instantly

What I Built

SaudaaAI is an end-to-end voice-to-quotation automation system designed for Indian commodity buyers. Instead of calling a salesperson or filling out a form, a buyer simply sends a voice message in Hindi or Gujarati on Telegram — and within seconds, they receive a professionally generated quotation document right in the chat. No app downloads, no portals, no friction.

The entire pipeline — from voice input to formatted document output — is fully automated using a custom n8n workflow connected to a Python backend I built and deployed on a cloud server.

The Problem It Solves

In India's commodity trading space, buyers are often non-tech-savvy, speak regional languages, and are used to placing orders verbally. Traditional quotation systems require them to navigate websites or call during business hours. SaudaaAI removes all of that — if you can send a WhatsApp voice note, you can get a quotation.

How I Built the Automation

The entire workflow is orchestrated using self-hosted n8n running on a Hetzner cloud server. Here is how each stage works:

1
Voice Input via Telegram
The workflow begins the moment a buyer sends a voice message to the Telegram bot. Telegram was chosen because it is widely used in India, works on low-end Android devices, and has native support for voice messages and file transfers.
2
Audio Retrieval
Telegram does not send the audio file directly in the webhook — it sends a file ID. The workflow makes a secondary API call to resolve this into a downloadable URL, then downloads the raw .ogg audio binary for processing.
3
Speech to Text — Sarvam AI
The audio is sent to Sarvam AI for transcription. Sarvam is an Indian AI company specifically trained on Indian languages including Hindi, Gujarati, and Marathi. Generic STT tools like OpenAI Whisper struggle with Indian accents, regional vocabulary, and code-switching. Sarvam handles all of this naturally — making it the right choice for a product built for Indian buyers.
4
AI Entity Extraction — Gemini
The raw transcription is fed into a Gemini AI Agent inside n8n. The agent extracts structured data — product names and quantities — from unstructured spoken text. For example, a buyer saying "mujhe pachas kilo gehun aur bees kilo chawal chahiye" gets converted into clean JSON: [{product: "wheat", qty: 50}, {product: "rice", qty: 20}]. Gemini was chosen for its strong multilingual understanding and ability to handle the natural ambiguity of spoken language.
5
Fuzzy Product Matching — RapidFuzz
Buyers do not say exact product names — they use shorthand, regional terms, or slight mispronunciations. A buyer might say "gehun" instead of "Wheat Premium Grade." I built a Python script using the RapidFuzz library that fuzzy-matches the extracted product name against a database of 500 products stored in Supabase. Even if the input is slightly off, it finds the closest real product and returns the correct price and SKU — making the system robust to the messiness of real spoken language.
6
Flask API on Hetzner
The fuzzy matching and quotation generation logic is wrapped into a Flask REST API deployed on the same Hetzner server where n8n is hosted. n8n calls localhost:5000/match and localhost:5000/generate — keeping latency near zero, avoiding external networking complexity, and keeping everything under one infrastructure roof. The Flask service runs as a permanent systemd service so it survives server restarts.
7
Quotation Generation — docxtpl
Once products are matched, the /generate endpoint fills a pre-designed Word document template using docxtpl — a Python library for templating DOCX files. The matched products, quantities, and prices are injected into the template, producing a clean, professional quotation document ready to be sent.
8
Telegram Document Reply
The generated DOCX is sent back to the user in the same Telegram chat where they sent the voice message. The full loop — voice in, quotation out — is complete.

Architecture

Buyer sends voice note on Telegram ↓ Telegram Trigger (n8n) ↓ Get File + Download Audio ↓ Sarvam AI — Speech to Text ↓ Gemini AI Agent — Extract Products & Quantities ↓ Flask /match — Fuzzy match against 500 products in Supabase ↓ Flask /generate — Fill DOCX quotation template ↓ Telegram — Send quotation document back to buyer

Why This Stack

Every tool in this stack was chosen deliberately. Sarvam AI for Indian language accuracy. Gemini for natural language understanding at scale. RapidFuzz for tolerating the messiness of real spoken input. Flask on the same server as n8n to eliminate any inter-service latency. Telegram as the delivery layer because it meets buyers where they already are.

The result is a system that feels like magic to the end user — speak naturally in your own language, get a quotation in seconds.
